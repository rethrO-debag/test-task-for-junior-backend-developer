# Task Service - сервис управления задачами с поддержкой периодичности

## Описание

Сервис предоставляет CRUD API для управления задачами с поддержкой различных типов периодичности. Подходит для использования в системах управления персоналом, где требуются повторяющиеся задачи (ежедневные обзвоны, ежемесячные отчеты, инвентаризации и т.д.).

## Требования

- Go 1.23+
- Docker и Docker Compose

## Быстрый запуск

```bash
docker compose up --build
```

После запуска сервис доступен по адресу `http://localhost:8080`.

При проблемах со схемой БД:
```bash
docker compose down -v
docker compose up --build
```

## Swagger

- UI: `http://localhost:8080/swagger/`
- OpenAPI JSON: `http://localhost:8080/swagger/openapi.json`

## API

Базовый префикс: `/api/v1`

|  Метод |    Эндпоинт   |        Описание        |
|--------|---------------|------------------------|
| POST   | `/tasks`      | Создание задачи        |
| GET    | `/tasks`      | Получение всех задач.  |
| GET    | `/tasks/{id}` | Получение задачи по ID |
| PUT    | `/tasks/{id}` | Обновление задачи      |
| DELETE | `/tasks/{id}` | Удаление задачи        |

## Периодические задачи

### Типы периодичности

|     Тип     |        Описание           |     Параметры       |
|-------------|---------------------------|---------------------|
| `daily`     | Каждые N дней             | `interval` (дни)    |
| `monthly`   | Определенные числа месяца | `month_days` (1-30) |
| `specific`  | Конкретные даты           | `specific_dates`    |
| `parity`    | Четные/нечетные дни       | `parity` (even/odd) |
| `none`      | Разовая задача            |          -          |

### Примеры

**Ежедневная (каждые 2 дня):**
```json
{
  "title": "Проверка серверов",
  "recurrence": {
    "type": "daily",
    "interval": 2,
    "start_date": "2026-04-22T00:00:00Z"
  }
}
```

**Ежемесячная (1 и 15 числа):**
```json
{
  "title": "Формирование отчета",
  "recurrence": {
    "type": "monthly",
    "month_days": [1, 15],
    "start_date": "2026-04-01T00:00:00Z"
  }
}
```

**На конкретные даты:**
```json
{
  "title": "Инвентаризация",
  "recurrence": {
    "type": "specific",
    "specific_dates": ["2026-05-10T00:00:00Z", "2026-05-20T00:00:00Z"],
    "start_date": "2026-05-10T00:00:00Z"
  }
}
```

**На четные дни:**
```json
{
  "title": "Обзвон клиентов",
  "recurrence": {
    "type": "parity",
    "parity": "even",
    "start_date": "2026-04-01T00:00:00Z"
  }
}
```

## Worker

Отдельный сервис для автоматической генерации экземпляров задач (запускается каждый час).

```bash
# Просмотр логов
docker compose logs worker
```

## Тестирование

Для тестирования API используйте тестовый скрипт:

```bash
# Сделать скрипт исполняемым
chmod +x test.sh

# Запустить тесты
./test.sh
```

Тестовый скрипт предоставляет интерактивное меню со следующими возможностями:
- Создание задач (автоматически или ручной ввод)
- Получение всех задач
- Получение задачи по ID
- Обновление задачи
- Удаление задачи
- Проверка worker
- Проверка экземпляров периодических задач
- Проверка обработки ошибок
- Полный автотест
- Очистка всех задач
- Информация о системе

## Архитектурные решения

- **JSONB поле `recurrence`** - гибкое хранение конфигурации без миграций
- **Отдельный worker** - не блокирует API, можно масштабировать
- **`parent_id` и `occurrence_date`** - связь шаблона с экземплярами

### Допущения
- Все даты в UTC
- Monthly: только дни 1-30
- Worker генерирует до 10 следующих экземпляров
- Интервал генерации: 1 час
